Documentation

For Version 4.0.x

This tool allows to perform Server Installations of LibreOffice without deeper knowledge about the WINDOWS installation process. A Server Installation is independent from the normal LibreOffice Installation and does not interact with that other LibreOffice.

The Server Installation can be used for tests with experimental LibreOffice versions and if you additionally need a particular LibreOffice Version because the feature is still only available in experimental versions or broken in your current normal LibreOffice installation.

The GUI Installer is not necessary for Installation of LibreOffice Master Versions from http://dev-builds.libreoffice.org/daily/master/Win-x86@6/current/ (and similar), these Master Versions will be installed independent from the normal stable LibreOffice version by default. But of course you also can install a Master Version (or stable version) using the GUI Installer, for example if you want that that version will not be overwritten by later (Master) installations.

System Requirements

Windows Installer 4.5 Microsoft .NET Framework 4 Client Profile (x86 und x64)

Installation

You can simply install the tool by a double click on the downloaded setup.exe in the WINDOWS File Explorer. If you doesn't have a prerequisite installed, it automatically will be installed before the installation of LibreOffice Server Install GUI starts

After Installation you will find  “LibreOffice Server Install GUI” in the subfolder "Reisisoft" in the WINDOWS Programs Start Center as well as on the desktop.

How To use:

1) Start the Program from WINDOWS Programs start center You can also use the GUI Installer to download a new LibreOffice Version (The latest release from the current and from the older branch can be downloaded. You can click on the buttons without starting the download. A prompt will show you, which LibreOffice version is going to get downloaded. If you have chosen “Download latest testing builds”, you have to select between the two versions. Click the button "Download helppack" beneath the button of the branch you want the helppack after choosing the language from the drop-down list beneath "Download latest master". There will also be a prompt. After download has finished, jump to step 4).

2) Click and browse for the installer.msi you downloaded. The name might be “libreoffice-3-6~2012-10-31_08.03.29_LibreOffice-Dev_3.6.4.0_Win_x86_install_en-US.msi” or similar. Select the installer with “Open Libreoffice installer”

3) Click and browse for a localized help installer, if available Select the installer with “Open LibreOffice help”

4a) Click to browse for the folder where you want to install your LibreOffice. It is not recommended to use the default WINDOWS Programs folder because administrator permissions. will be required. Use something like “C:\LibreOffice_Server\LibreOfficeVersionxx\”

4b) Select the folder, in whose subfolder you want to install LibreOffice parallel. Tick the checkbox "Should a subfolder be created automatically" and enter the name of the subfolder in the text field below. If you download LibreOffice via this program, a name for the subfolder automatically will be suggested. If you don't want to have a subfolder created, simply uncheck the ckeckbox mentioned above.

5) Click the button “Start installation”: The normal LibreOffice Installation dialogs will appear and show you the process of the installation.

6) When Installation has finished, click and modify it due to your needs. All over the program should be tooltips available, which explain some features a little bit more detailed. If you have finished editing, the bootstrap.ini, which is highly recommended, simply press “Save bootstrap.ini”. (Details about the bootstrap.ini can be found underneath)

6b) If you want to have a desktop icon for the previously created LibreOffice server installation, click the button “Create shortcut” after the installation started.

7) Ready, you can close the GUI Installer and start you new LibreOffice server installation by a double click on “soffice.exe” in the “program” subfolder of the folder you selected in step 4, or via the desktop shortcut you created manually.

bootstrap.ini

The input pane allows to define what user profile the server installation of LibreOffice will use. Details concerning User Profile you find here:https://wiki.documentfoundation.org/UserProfile#User_profile_location.

There are 4 appropriate places for your user profile:

a) You change nothing, your Server installation will use (and create, if necessary) the default profile defined by the default entry in bootstrap.ini like “UserInstallation=$SYSUSERCONFIG/LOdev/4”

b) You use your normal user profile with input "UserInstallation=$SYSUSERCONFIG/LibreOffice/4" (for LibreOffice Version 4.x). That's risky, your User Profile might become unusable

c) you use a separate User Profile only for this version with entry "UserInstallation=$ORIGIN/.."

d) You use a separate common user profile for your experimental Server Installations with entries like "UserInstallation=$SYSUSERCONFIG/LibreOffice-dev/3" or "UserInstallation=$SYSUSERCONFIG/LibreOffice/3experimental" or similar

e) You may use the snipped provided right beneath the textbox in order to not override your profile

Manager

Clicking the manager button will open a new manager window. In the manager you can Add or Delete existing parallel installations.

Add

Inside the manager, click "Manually add parallel installation". A dialogue will pop up. Click "Configure path" and select the soffice.exe of the parallel installation. Confirm by clicking Okay.

Delete

Check all list parallel installation you want to delete. After that, click "Delete selected". After the command line disappears, the parallel installation has been finished. Attention: Deleting files is always risky!

Caveat emptor, you do all this on your own risk, but it should be working without any problem...
